\section{Trapezoidal velocity profile}

The first trajectory type that has been implemented is the trapezoidal velocity profile trajectory. It is possible to limit velocity and acceleration, but not the jerk. Using the acceleration and velocity constraints that were given in the task, the synchronized velocity profile has been calculated in the file \textit{jspace\_trapz.m}. The outputs are time stamps and corresponding velocity values, which are then imported into Simulink using a "From Workspace" block and fed into an integrator to receive the needed position information. As the simulation uses the vectors from the workspace, the script \textit{initialize.m} needs to be run before the simulation.

The trajectory itself, with position, velocity and acceleration profile can be viewed in figure \ref{fig:jspace_trapz_traj}.

\begin{figure}[H]

The maximum velocity should not exceed $0.05\,\si{\meter\per\second}$, while the acceleration should be capped at $0.1\,\si{\meter\per\square\second}$. It is apparent, that none of the velocity or acceleration constraints are violated.

\section{Double-S velocity profile}
The other type of joint space trajectory that has been implemented is the double-S velocity profile, which has the additional capability to also limit the jerk to $5\,\si{\meter\per\cubic\second}$ by using a trapezoidal acceleration profile. The calculation is implemented in the function \textit{jspace\_double\_s.m}. It is also necessary to run \textit{initialize.m} before starting the simulation.

\begin{figure}[H]
	\centering
	\includegraphics[width=1.0\linewidth]{cus_imgs/joint_double_s.pdf}
	\caption{Double-S trajectory, including position, velocity, acceleration and jerk profiles in joint space}
	\label{fig:jspace_ds_traj}
\end{figure}

Because of the relatively high jerk limit, the double-S shape of the joint velocity is not perfectly visible, but still there.
